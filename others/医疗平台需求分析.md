# 医疗平台需求分析（精简版）

## 1. 项目目标与范围

| 类别 | 内容 | 说明 |
|---|---|---|
| 项目目标 | 构建智能医疗平台MVP | 以“AI建议 + 医生确认”为核心模式 |
| 智能能力 | 接入 DeepSeek API | 支持问答、摘要、结构化抽取、分诊建议 |
| In Scope | 智能问答、分诊建议、病历结构化、随访记录、医生工作台、审计日志 | 首期聚焦文本类智能能力 |
| Out of Scope | 自动诊断闭环、自动开药、替代HIS/EMR核心系统、影像算法自研 | 高风险场景不做自动决策 |

## 2. 角色与核心需求

| 角色 | 核心需求 | 系统能力 | 优先级 |
|---|---|---|---|
| 患者 | 就诊前咨询、流程指引、复诊提醒 | 智能问答、导诊建议、消息提醒 | P0 |
| 医生 | 快速获取病历摘要与建议 | 摘要生成、结构化抽取、建议复核 | P0 |
| 运营/管理员 | 可监管、可追溯、可配置 | 审计日志、权限管理、知识库维护、成本看板 | P1 |

## 3. 功能需求清单（MVP）

| 模块 | 需求项 | 说明 | 优先级 | 验收标准（简） |
|---|---|---|---|---|
| AI接入层 | DeepSeek API网关 | 鉴权、限流、重试、超时控制 | P0 | 成功率≥99%，异常可回退 |
| 智能问答 | 医疗常见问答 | 基于知识库RAG返回可解释答案 | P0 | 关键问答准确率≥85% |
| 分诊建议 | 科室/就诊建议 | 输出“建议级”结果并标注非诊断 | P0 | 高风险问题强制人工确认 |
| 病历处理 | 摘要与字段抽取 | 从文本中抽取主诉、现病史等结构化字段 | P0 | 结构化抽取F1≥0.90 |
| 医生工作台 | 建议复核与修订 | 展示建议来源、支持一键编辑确认 | P0 | 医生确认链路完整可追溯 |
| 安全合规 | 脱敏与访问控制 | 数据分级、最小权限、操作留痕 | P0 | 敏感数据脱敏覆盖率100% |
| 运营看板 | 成本与质量监控 | 调用量、耗时、失败率、费用统计 | P1 | 日/周报可导出 |

## 4. 非功能需求

| 类别 | 指标要求 | 说明 |
|---|---|---|
| 性能 | P95响应时间≤3秒 | 常规问答场景 |
| 可用性 | 核心服务可用性≥99.9% | 含降级策略 |
| 安全 | 传输与存储加密、密钥轮换 | API密钥集中托管 |
| 审计 | 全链路日志可追溯 | 包含模型版本、提示词版本、操作者 |
| 合规 | 遵循个人信息与医疗数据相关法规 | 默认最小化采集与可撤回 |

## 5. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| 模型幻觉 | 误导医疗建议 | RAG + 规则兜底 + 医生复核 |
| 合规违规 | 法律与声誉风险 | 脱敏、权限分层、审计与合规评审 |
| 成本失控 | 预算超支 | 限流、缓存、分级模型策略 |
| 系统依赖外部API | 稳定性下降 | 超时重试、熔断、降级到规则引擎 |

## 6. 验收标准（总览）

| 维度 | 验收标准 |
|---|---|
| 功能完整性 | P0需求全部上线并通过UAT |
| 准确性 | 结构化抽取F1≥0.90，关键问答准确率≥85% |
| 安全合规 | 敏感数据脱敏覆盖100%，高风险回答人工确认率100% |
| 可追溯性 | 每次AI输出具备输入、来源、版本、操作者日志 |
| 性能与稳定性 | P95≤3秒，核心服务可用性≥99.9% |

## 7. 待确认项（最小集合）

| 序号 | 待确认问题 | 影响范围 |
|---|---|---|
| 1 | 首期优先场景（患者端问答/医生端病历处理/随访） | 决定MVP功能取舍 |
| 2 | 部署方式（公有云/私有化/混合） | 影响架构与合规成本 |
| 3 | 对接系统范围（HIS/EMR/CRM） | 影响接口与实施周期 |
