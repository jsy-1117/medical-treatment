# 智慧医疗平台 — 项目介绍

## 项目概述

智慧医疗平台是一个基于 Spring Boot 3 + Vue 3 的**全栈 AI 医疗管理系统**，覆盖患者、医生、管理员三角色完整业务闭环。平台以 **DeepSeek-V4-Flash 大语言模型**为核心引擎，实现智能导诊、AI 辅助诊断、流式对话等前沿能力，区别于传统 CRUD 医疗项目。

- **后端**：Java 21 · Spring Boot 3.2.5 · MyBatis-Plus · MySQL 8 · Redis · JWT
- **前端**：Vue 3 · TypeScript · Vite · Element Plus · Pinia · SCSS
- **AI**：DeepSeek-V4-Flash API · SSE 流式响应 · 结构化 Prompt Engineering

## 项目亮点

### 1. LLM 智能导诊（患者端）

浮动聊天窗口嵌入患者全流程页面，实时与 AI 对话：

- **症状 → 推荐科室 → 一键挂号**的全闭环体验
- 患者输入症状描述，AI 结合院内**实时科室数据、医生坐诊信息、可挂号源**给出精准推荐
- 推荐科室卡片支持**点击直达预约页面**，自动筛选对应科室出诊医生
- 基于 DeepSeek-V4-Flash 1M 上下文窗口，可一次性注入全院信息而不截断
- 紧急症状检测：AI 识别关键词自动弹窗提醒尽快就医

> **区别于其他项目：** 不是简单的"问 AI → 显示文字"，而是将 AI 输出与业务系统**深度打通**——推荐结果直接驱动挂号流程，形成从咨询到行动的真实闭环。

### 2. AI 辅助诊断（医生端）

医生接诊时，一键调用 AI 生成专业诊断建议：

- 根据患者**症状 + 基本信息（年龄/性别）+ 历史诊断记录**综合分析
- 用**自然人语言**输出结构化建议：【初步诊断】【鉴别诊断】【处方建议】【检查建议】【注意事项】
- 黄色警告卡片明确标注"仅供参考"，强调医生最终决策权
- 独立医生专用 Prompt（温度 0.3 确保输出稳定性），与患者端导诊 Prompt 完全分离
- 集成于医生工作站现存诊断对话框中，无侵入式 UI 改动

> **区别于其他项目：** AI 不是独立功能模块，而是**深度嵌入医生工作流**——在填写诊断表单的关键环节主动提供辅助，降低医患沟通成本与误诊风险。

### 3. SSE 流式对话引擎（自研）

完整实现从 DeepSeek API 到前端 UI 的**逐字流式输出**：

- **后端**：基于 Spring MVC `SseEmitter` 零额外依赖实现 SSE 推送，后台线程逐块解析 DeepSeek 流式响应
- **前端**：原生 `fetch()` + `ReadableStream` 解析，绕过 Axios 限制，实现逐字打字机效果
- **多层容错**：流式失败自动降级到同步模式 + 指数退避重试 + 30s 超时保护
- **线程安全**：DB 查询前置到 Tomcat 线程执行，规避 Spring DevTools ClassLoader 冲突
- **上下文管理**：动态上下文（科室/医生/排班）超过 2000 字符自动截断，防止 Prompt 超长

> **区别于其他项目：** 大多数全栈项目的 AI 集成是"等几秒出全文"，本项目实现了**工业级流式推送**——用户看到 AI 边思考边写，体验显著优于同步等待。

### 4. 动态排序与 SQL 注入防护

排班查询支持多字段动态排序，用**白名单 + Lambda 映射**策略防注入：

- 前端传入 `sortField=workDate,shiftType&sortDirection=asc,desc`
- 后端维护 **白名单 Map**（字段名 → Lambda getter），不在白名单的字段静默忽略
- 方向数量不足时默认补齐 ASC，保证不会因参数不对称抛出异常
- 此方案优于字符串拼接 ORDER BY（有注入风险）和 MyBatis 动态 SQL（易出错）

> **区别于其他项目：** 不是简单的外键查询，而是**在安全前提下实现了灵活的动态排序**，单元测试覆盖 8 种边界场景（包括 `DROP TABLE` 注入测试）。

### 5. 三角色权限体系 + JWT 无状态认证

- 三种 Token（`patient_token` / `doctor_token` / `admin_token`）独立管理与路由守卫
- Spring Security 过滤器链按角色 + 路径精确分级：公开接口（`/api/public/**`）、患者接口（`/api/patient/**`）、医生接口（`/api/doctor-portal/**`）、管理接口（`/api/admin/**`）、AI 接口（`/api/ai/**` 全部公开）
- 医生诊断建议接口挂在 `/api/doctor-portal/**` 下，区别于公开的 AI 端点，确保患者隐私数据安全

### 6. 完整测试体系

- **41 个后端集成测试用例**，遵循华为 CodeArts TestPlan 自动化测试规范
- 按模块分目录组织（`BackendTest/auth/`、`/department/`、`/doctor/`、`/schedule/`、`/appointment/`、`/ai/`）
- 使用 H2 内存数据库（MySQL 兼容模式）+ Mock Redis 实现全隔离测试环境
- 覆盖 Controller 层（MockMvc）+ Service 层（`@SpringBootTest`）+ 工具方法（反射调用私有方法）
- 流式端点测试覆盖 SSE Content-Type、超时回调、参数校验、故障异常、同步端点不受影响

### 7. 生产级工程细节

- **ClassLoader 兼容修复**：识别并解决 Spring DevTools 在 ForkJoinPool 线程中的 `ClassCastException`——将 DB 查询提至 Tomcat 线程执行
- **SSE 协议正确性**：多行 `data:` 字段用 `+=` 拼接而非 `=` 覆盖，符合 W3C SSE 规范
- **用户取消处理**：组件卸载时 AbortController 取消流，通过 `onCancel` 回调让 Promise 干净退出，避免内存泄漏
- **Promise 生命周期**：流结束但未发 `done` 事件时（如 `finish_reason: length`），后端补发 fallback `done` 事件，防止前端永久 loading

### 8. 组件化与 UI 设计

- 患者端：统一 `PatientLayout` 布局（导航栏 + 内容区），蓝色医疗主题
- 医生端：独立 `DoctorLayout`，紫色系主题区分角色
- 智能导诊 `ChatWindow.vue`：浮动聊天按钮（`fixed + z-index: 2000`），可快速移植到任意页面
- 响应式设计：Element Plus 组件库 + SCSS 变量体系，全平台视觉统一
